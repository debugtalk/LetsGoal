### 实施阶段

**阶段 1：入口**（Skill 触发 + 飞书文档需求流）

**Skill 触发机制**：使用 Claude Code 标准 skill 安装方式（`npx skills add`），用户在任意项目中输入 `/letsgoal` 即可激活。SKILL.md 定义 `/letsgoal` 触发条件和交互流程。

**飞书文档需求流**：用户通过 `/letsgoal` 输入需求内容（概括性描述），系统基于用户输入生成飞书文档，按照 request.md 格式进行结构化 + 需求补充，给到用户确认。后续开发迭代过程中，每轮关键进展和结论以 append 方式追加到同一份飞书文档。

**阶段 2：可观测性**（决策通知 + 迭代过程记录）

**关键决策通知**：终端通知为默认通道（零配置），飞书消息通知通过 `lark-cli im +messages-send` 可选。通知事件包括：escalation（始终通知）、连续同类失败（同一 category 连续 3 次）、任务完成。通知配置通过 `LoopConfig.notify_channel` 控制。

**迭代过程记录**：每一轮迭代的关键进展和结论以 append 方式写入阶段 1 的飞书文档，只追加不修改。本地 `.letsgoal/` 仍保留完整记录作为数据源。

**阶段 3：质量**（软分补全 + L0-L3 分层 + 归因增强）

**软分维度补全**：复用已有 lint 输出实现 smells（warning 计数归一化）和 complexity（eslint 规则或 changed files 启发式）；docs 检查是否有文档文件变更。

**L0-L3 分层评估**：重构 `evaluateTask()` 为 L0（lint+typecheck）→ L1（test）→ L2（coverage+soft）→ L3（skill），每层失败短路。executor prompt 根据失败层级给 Claude 不同聚焦指引。

**归因分类器增强**：新增 6 条规则（跨迭代模式检测 + 单迭代增强规则），`classifyFailure` 增加 `iterationHistory` 可选参数。

### 关键设计决策

1. **Skill 安装**：使用 Claude Code 标准 skill 安装方式，不做自定义安装/卸载脚本
2. **用户输入 + 系统结构化**：用户只需概括性描述需求，系统负责结构化和补充，生成飞书文档供确认
3. **飞书文档贯穿全程**：系统创建飞书文档承载需求 → 拆解确认 → 迭代记录，只追加不修改
4. **lark-cli 接口**：`docs +create --title --markdown` 建文档、`docs +update --doc --mode append --markdown` 追加、`im +messages-send --chat-id --markdown` 发消息
